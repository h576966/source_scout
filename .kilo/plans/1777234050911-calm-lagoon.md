### S3: Remove unnecessary `replace("Z", "+00:00")` and extract shared `_parse_pushed_at`

**Files:** `src/repo_finder/ranker.py`, `src/repo_finder/repo_inspector.py`

Python 3.11+ handles `Z` suffix natively in `datetime.fromisoformat()`. The `replace("Z", "+00:00")` call is dead work. Additionally, the same try/except `fromisoformat` pattern appears 3 times. Extract:

```python
# In repo_inspector.py (already the natural home for repo inspection helpers)
def _parse_pushed_at(pushed_at: str | None) -> datetime | None:
    if not pushed_at:
        return None
    try:
        return datetime.fromisoformat(pushed_at)
    except (ValueError, AttributeError):
        return None
```

Update `score_activity`, `_identify_risks`, and `_evaluate_activity` to call this helper.

def _now_iso() -> str:

Replace all 9 inline `datetime.now(UTC).isoformat()` expressions. This is a 1-line change per site, net-negative lines.

### S6: Remove dead `TypeError` catch in `_get_weight`

**File:** `src/repo_finder/ranker.py:145-156`

`float(x)` on a string from `os.environ.get(key, str(default))` will never raise `TypeError` (strings are valid input to `float()`). Remove the `TypeError` from the except clause:

```python
def _get_weight(key: str, default: float) -> float:
    try:
        return float(os.environ.get(key, str(default)))
    except ValueError:
        return default
```

### S7: Merge `pattern_library.py` into the DuckDB cache

**Files:** `src/repo_finder/pattern_library.py` (delete), `src/repo_finder/cache.py`, `src/repo_finder/pattern_extractor.py`

The pattern library is a 31-line in-memory LRU cache with `asyncio.Lock`. It serves one caller: `extract_patterns_deep`. The DuckDB cache (`cache.py`) already provides TTL-based key-value storage.

**Option A (simplest):** Delete `pattern_library.py`, inline a simple module-level dict in `pattern_extractor.py` (no LRU, no lock). For an MCP server processing one tool at a time, the lock provides no value.

**Option B (more cohesive):** Store `DeepPatternReport` in the DuckDB cache via `cache_get`/`cache_set`. This gives persistence and TTL expiry for free.

**Decision:** Option A. A module-level dict is sufficient. The async lock was premature optimization. Removing the entire 31-line module and replacing with a 2-line dict in `pattern_extractor.py` is a net negative on lines of code.

### S8: Create `tests/test_server.py`

### S9: Unify verdict thresholding (documentation only)

The 4 verdict algorithms serve different domains (search ranking, inspection quality, pattern richness, deep extraction) and should NOT be merged. Instead, document each in the module where it lives with a comment explaining the domain:

```python
# Verdict: search-space relevance based on metadata signals
if total >= 0.70:
    verdict = "useful"

# Verdict: after full inspection — archive, quality signals
if quality.score >= 0.70:
    return "useful", "..."

# Verdict: pattern extraction — richness of extractable patterns
verdict = "useful" if len(patterns) >= 2 else ...
```

No logic change — just a comment per verdict block explaining its domain.

### S10: Remove duplicated local `datetime` imports

**Files:** `src/repo_finder/server.py:52-55`, `src/repo_finder/repo_inspector.py:175`

Both files do `from datetime import datetime as dt` inside a function despite already importing `datetime` at the top. Since `datetime` is imported as `datetime` (not `from datetime import datetime`), there's no shadowing issue. The local imports can simply be removed, using the top-level `datetime.now(UTC)` directly.

Wait — `server.py` imports `from datetime import UTC, datetime` at line 2. So `datetime` IS the class, not the module. This means the local `from datetime import datetime as dt` is creating the same alias — it's just an unnecessary re-import. Remove it, use `datetime.now(UTC)` directly.
